# WPF 工具类应用开发规则 (Solo 模式优化)

## 核心目标
- **极速交付**: 减少过度设计，优先实现核心功能。
- **轻量化**: 尽量减少外部依赖，除非是 UI 美化或核心逻辑必须。

## 架构与实现
- **简易 MVVM**: 对于简单工具，可以直接在 ViewModel 中处理逻辑，无需过度分层。
- **UI 库**: 默认使用 `HandyControl` 或 `MaterialDesignInXaml`，Trae 应优先使用这些库的内置控件。
- **数据存储**: 优先使用 `LiteDB` 或 `Json` 文件进行本地配置存储，避免复杂的数据库安装。

## Trae Solo 模式指令
- "创建一个单窗口工具，包含 [功能描述]，使用 HandyControl 风格。"
- "为该工具添加配置保存功能，使用 Json 格式存储在 AppData 目录下。"

## 调试偏好
- 必须包含一个 `.NET Core 控制台` 项目用于快速验证核心算法，避免频繁启动 GUI 进行调试。
